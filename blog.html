设计作品中的“廉价感”是如何产生的？

2017-01-20

什么是贵，什么是廉价？这个感受虽然是根据每个人的生活经历以及见识所决定。但同时，更是被一种常规的社会认知而决定的。高档与廉价的感觉是通过和同品类的比较，以及和我们心智中对于事物认知的对比而得来的。

  • 该事物具备了与最主要功能特性极度不合的成分 印象中基本屬性≠看到事物最主要特性
  • 距离感
  • 对身边每日接触到的事物产生各方面的疲劳感，这样的疲劳感虽然不会直接的带来“廉价”这个感觉。但是一旦出现了一个基本功能属性与之完全一致的产品时，我们会因为对旧事物的疲劳感，而产生“廉价”的感觉。

造成廉价原因总结： 用没用料，决定了是否便宜；用没用心，决定了是否廉价；

設計構圖

2017-01-20

構圖作用：為了加強視覺的張力、增加設計的趣味性，吸引并引導觀者的注意力。

視覺焦點

不同視角構圖：

  1. 平行-->常見/平靜，無聊、單調、習以為常
  2. 俯視-->廣闊/渺小/強大威嚴，注重元素擺放位置、組合的形狀，常見視角俯視、以高視角俯視地面
  3. 仰視-->高大/積極向上，以比人小的視角去觀看物體
  4. 第一視角-->帶入其中
  5. 傾斜構圖-->打破平衡感、穩定感
  6. 近距離視覺-->注意力集中在視覺主題對象，減少不必要的干擾
  7. 反轉構圖(倒著看)

違背常規視覺的對象在畫面當中就可以匯成視覺焦點，吸引眼球。

設計師要有勇氣去打破常規，而不是強調還原真實。人看到的東西都會不自視的存在記憶當中，而使用常見的視覺構圖方法則無法滿足人的好奇心。

Git_版本控制

2017-01-20

  • window+R cmd cd Desktop 進入子資料夾
  • cd.. 回父資料夾
  • cd/ 回根資料夾
  • dir 顯示所有資料
  • git init 資料初始化
  • git status 檢查狀態，紅色代表還沒上傳集結區，綠色代表已上傳
  • git add .全部加入集結區
  • git commit -m "版本號（內容隨意）"
  • git remote add origin （貼上鏈接），連結遠端
  • git push origin master 資料推入雲端
  • git clone（貼上鏈接），下載版本
  • git pull，拉下雲端（本地和雲端不一致時用）
  • git log，查看歷史版本
  • git reset --hard （版本號前6碼），檔案恢復
